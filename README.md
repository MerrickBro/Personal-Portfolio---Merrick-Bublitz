# Personal Portfolio Project

For your last project as part of the web unit, you are to build a personal portfolio.

Your portfolio should represent you as a student, future employee, programmer, designer, or professional. Your goal is to create a clean, organized, and professional webpage that could be shown to another teacher, employer, advisory member, of a future client.

This project does not need to be perfect, one's personal portfolio is always evolving and never reaches its final form until retirement.

Just like our previous project, you will be building this as a single page website, unless you want to go above and beyond. You will be required to create and submit a wireframe of your website here on Canvas, then build your website through the Github Classroom link.

A good developer portfolio should contain the following sections:

## Header / Hero

This will be the first section that people see when opening your site. Use this section as your first chance to sell yourself, give the individual reading this a reason to scroll through the rest.

### Your hero should include the following

- Your name
- A short title or description of yourself
- EX: "Computer Programming Student"
- EX: "Future Software Developer"
- A short introduction sentence or paragraph
- Optional: A photo, avatar, logo, or other visual design element.
  Remember: This site should be professional, and presentable to future employers.

## About Me

This section should explain who you are and what you're interested in.

### It should include the following

- A short paragraph about yourself
- Your interests in technology, programming, business, design, cybersecurity, robotics, or another related area.
- Your future goals.

## Skills / Certifications

Create a section that lists any skill you have learned or are currently learning, if you have certifications, this is a good place to include those as well.

You can design this section however you like, including but not limited to a list, cards, badges, or icons.

## Projects

This section should show at minimum 2 projects that you have worked on. These can be independent projects, or they could be assignments that you have worked on in class that you are particularly proud of.

### Each project should include

- Project name
- Short description of the project
- Tools or languages used
- What you learned/contributed

## Experience

This section should include work experience, volunteer experience, leadership experience, school activities, or other relevant items,

### Examples include

- Jobs
- Internships
- Volunteer work
- BPA
- FIRST/VEX Robotics

### Every experience should include

- Role or position
- Organization or location
- Approx. dates if known
- A short description of what you did

## Contact

Your website should end with a simple contact section. You do not need to include your personal phone number or email.

### Examples of what to include

- Github profile
- LinkedIn profile
- A fake contact section for design purposes.

## Technical Requirements

### Your project must include

- Minimum 1 html file
- Minimum 1 css file
- Minimum 1 javascript file
- Proper HTML structure
- A meaningful page title
- The above sections
- Navigation links at the top of the page
- Professional CSS styling
- At least 1 Javascript feature
- Clean indentation and organized code

## Javascript Requirement

Your site must include at lease one Javascript feature.

### Examples

- A button that shows a message
- A project details button that shows/hides more information
- A random quote or fun fact button
  Your Javascript does not need to be complicated. It just needs to work and be connected correctly to your webpage.

## Design Expectations

Your portfolio should look professional and easy to read.

### Your site should include

- Consistent colors
- Readable fonts
- Good spacing
- Organized sections
- Clear headings
- Buttons or links that are styled
